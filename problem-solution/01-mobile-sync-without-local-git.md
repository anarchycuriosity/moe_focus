# 01 - 移动端为什么不用本地 Git 命令同步

## 问题背景

桌面端 MoeFocus 是 Electron 应用，可以在用户数据目录里初始化 Git 仓库，然后执行 `commit`、`pull`、`push`。移动端是 Expo / React Native 应用，它运行在手机系统沙盒里，不能像桌面端那样稳定调用本机 `git` 命令。

旧移动端代码只有“导出同步数据”的提示，并没有真正把数据同步到远程仓库。这会导致手机端和桌面端形成两个孤岛。

## 约束分析

1. 手机端不能依赖系统已经安装 Git。
2. Expo 托管环境不适合维护一个完整的本地 Git 工作区。
3. 桌面端已经约定了同步文件格式：`data/*.json` 和 `sums/*.md`。
4. 同步必须避免重复导入同一条专注记录，否则统计时间会翻倍。

## 解决方案

移动端改用 GitHub Contents API：

1. 用户在设置页填写仓库地址、分支和 GitHub Token。
2. 手机端读取远程 `data/focus_sessions.json` 与 `data/long_term_goals.json`。
3. 本地数据按 `uuid` 导出为对象。
4. 远程对象与本地对象做并集合并。
5. 合并后的 JSON 写回 GitHub。
6. 手机端根据本地专注记录生成 `sums/YYYY-MM-DD.md` 并上传。

## 为什么必须使用 UUID

数据库自增 ID 只在单台设备上有意义。桌面端的第 3 条专注记录和手机端的第 3 条专注记录，不一定是同一件事。

因此同步文件使用 `uuid` 作为键：

```json
{
  "session-uuid": {
    "subject": "学习网络编程",
    "actual_duration_sec": 1500,
    "date": "2026-06-08"
  }
}
```

这样重复同步时，同一个 UUID 只会导入一次。

## 当前边界

当前移动端同步覆盖：

1. 专注会话。
2. 长期任务。
3. 日记 Markdown。

任务库和每日 TODO 目前仍偏本地化，因为桌面端同步格式尚未导出这两类表。后续如果要做到完全跨设备任务列表同步，需要先给桌面端补充 `tasks.json` 与 `todo_items.json` 的导出导入协议。
