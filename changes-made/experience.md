# MoeFocus 修复经验索引

## 01 后台专注计时异常

问题类型：前端计时、事件循环、后台节流。

核心经验：倒计时不能把 `setInterval` 当作可靠时钟。可靠做法是记录阶段结束的绝对时间，再用当前时间计算剩余秒数。

记录文档：`01_background_focus_timer.md`

## 02 暂停后继续导致进度条重置

问题类型：状态建模、UI 显示状态、统计记录状态。

核心经验：同一个变量不要同时承担 UI 总进度和数据库分段统计基准。继续计时时，应保留 UI 总时长，同时单独记录当前数据库会话的开始剩余秒数。

记录文档：`02_resume_progress_display.md`

## 03 点击暂停后无法暂停

问题类型：React 组件实例、hook 状态隔离、定时器句柄生命周期。

核心经验：如果多个页面都能控制同一个计时器，interval 句柄不能放在单个组件实例的 `useRef` 中。应提升到模块级或统一 store，保证开始、暂停、继续操作的是同一个真实计时器。

记录文档：`03_pause_button_not_working.md`
