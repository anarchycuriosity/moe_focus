# 03 - 暗色模式字体对比：新增模块为什么看起来不清楚

## 一、问题背景

用户反馈：暗色模式下，Codex 后面新增的模块字体不够显眼，观感不好。

本次主要影响这些新增区域：

1. 长期任务页面。
2. 侧边栏同步反馈。
3. 统计页同步反馈。
4. 日记生成确认弹窗。

这些区域共同的问题是：部分样式直接写了 `white` 或 `rgba(255, 255, 255, ...)` 作为背景，但文字仍然使用暗色主题变量 `--moe-text`。在暗色主题下，这会造成背景、文字、边框之间的层次不稳定。

## 二、前置知识

### 2.1 主题变量

主题变量是 CSS 自定义属性。例如：

```css
:root {
  --moe-text: #E8E4F0;
  --moe-bg: #1A1A2E;
  --moe-glass-bg: rgba(30, 30, 56, 0.65);
}
```

暗色模式下，文字通常应该更亮，背景应该更深。浅色模式下则相反。

### 2.2 为什么不能随手写 white

`white` 在浅色模式下可能没问题，但在暗色模式下会变成一个强烈发光的色块。如果此时文字仍然是浅色，就会出现“亮底浅字”的问题。

错误示例：

```css
.task_row {
  background: rgba(255, 255, 255, 0.48);
  color: var(--moe-text);
}
```

在暗色主题中，`--moe-text` 是浅色。浅色文字放在半透明白底上，可读性会下降。

### 2.3 字重也是对比的一部分

对比不只由颜色决定，也由字体大小和字重决定。

辅助文字如果太细、太灰，即使颜色理论上可见，也会显得“飘”。因此这次对新增模块的说明文字、按钮文字、状态标签都适当提高了 `font-weight`。

## 三、这次的解决策略

### 3.1 暗色模式默认使用深色底

长期任务列表行从浅白底改成深色透明底：

```css
.task_row {
  background: rgba(255, 255, 255, 0.08);
}
```

这不是纯黑，而是轻微发亮的玻璃层，能和整体 UI 保持一致。

### 3.2 浅色主题单独覆盖

浅色模式仍然可以使用白底：

```css
[data-theme] .task_row {
  background: rgba(255, 255, 255, 0.58);
}
```

这样暗色和浅色不互相拖累。

### 3.3 状态标签改成暗色可读颜色

暗色模式下，绿色、黄色、红色标签使用更亮的前景色；浅色模式再覆盖回更深的颜色。

这能保证：

- 暗色模式：颜色更亮，容易识别。
- 浅色模式：颜色更深，不会刺眼。

## 五、项目作业

找一个自己写过的网页，完成一次暗色模式审查：

1. 搜索所有 `white`、`#fff`、`rgba(255`。
2. 判断这些颜色是否在暗色模式下仍然合理。
3. 对卡片、弹窗、输入框、按钮分别写暗色和浅色覆盖。
4. 用截图对比修改前后。

进阶要求：为项目整理一份颜色变量表，禁止业务组件随手写固定背景色。
